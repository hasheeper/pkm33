# Commander System & EVO Refresh – 2026-02-05

## 概述
本次更新聚焦于 Commander System V2、Smart Bubble 悬浮窗与 EVO（生命进化/羁绊共鸣）交互体验，目标是统一“装填式”机制、消除误触及刷新异常，并对初始冷却与解锁条件进行更清晰的分级。

## 关键改动

### 1. Smart Bubble 悬浮窗
- **层级调整**：`z-index` 下调至 900，保证结算滤罩出现时悬浮窗会被遮挡，解决奖励画面穿插问题。
- **EVO 提示刷新**：当检测到“生命进化”或“羁绊共鸣”提示日志时立即刷新悬浮窗，视觉与日志同步。
- **解锁/冷却态一致**：
  - `enable_styles = false` 时，风格条目完全隐藏，不再显示冷却状态。
  - Commander 设置中 `enableAVS` 或 `enableCommander` 任意关闭时，TACTIC 面板不可见。
  - 冷却期内的风格/指令不再参与轮播，避免“休息中仍出现”的困扰。

### 2. Commander System V2
- **初始冷却规则**：
  - 同步率 < 120：进入战斗即设定冷却，无法首回合直接指挥。
  - 同步率 ≥ 120：无初始冷却，开局即可使用。
- **使用后冷却**：仍依同步率动态设定，但确保冷却不会被强制压成 1 回合。
- **可用性判定**：
  - 训练家同步率 < 60 时，Commander System 被视为“默契不足”，直接不可用。
  - TACTIC 菜单在 UI 中会显示“默契不足”提示，帮助判断解锁条件。

### 3. 古武 Styles 系统
- **初始冷却门槛**：熟练度 < 101 的训练家，进场即进入初始休憩；101 及以上可首回合使用风格。
- **互斥逻辑**：与 Commander 指令、EVO 装填保持互斥关系，保证同一回合只能武装一种机制。

### 4. EVO 触发流程
- **装填式改造**：生命进化（bio）与羁绊共鸣（bond）纳入“装填→回合结算触发”流程，并与风格/指令互斥。
- **日志纠正**：针对 `Bond Evolution` 触发显示错误的文案问题添加映射，确保“羁绊共鸣”展示正确标签。
- **回合结束刷新**：EVO 动画结束后强制刷新 Smart Bubble，及时反映锁定状态解除。

### 5. 其他 UI 修复
- 点击 **FIGHT** 不再重置已锁定的悬浮窗状态，防止玩家误以为指令被取消。
- Commander Styles 菜单在休息阶段显示明确文案（“休憩中：X回合”“不可用”），避免冷却与未解锁状态混淆。

## 使用建议
1. **测试解锁条件**：在 `systems/data-loader.js` 中调节 `player.unlocks` 与 `settings`，可快速验证指令/风格是否正确隐藏。
2. **观察日志**：Commander 与 EVO 均会输出彩色提示，可从控制台或战报确认是否在正确阶段刷新。
3. **对战体验**：建议使用高熟练度/高同步率档案体验零初始冷却，与低熟练度对比差异。
